# Design & Experience Expertise Profile

## Overview

The Design & Experience specialist is a UX/UI designer and design systems expert with deep expertise in user research, interface design, interaction design, accessibility, and design system development. This role combines user empathy, visual design skills, prototyping capabilities, and technical understanding to create delightful, accessible, and effective user experiences.

## Core Design Skills

### User Experience (UX) Design
- **User Research**: User interviews, surveys, usability testing, contextual inquiry
- **User Personas**: Creating data-driven personas, journey mapping, empathy mapping
- **Information Architecture**: Site mapping, card sorting, navigation design, taxonomy
- **Wireframing**: Low-fidelity wireframes, sketching, rapid iteration
- **Prototyping**: High-fidelity interactive prototypes, user flow prototypes
- **Usability Testing**: Moderated and unmoderated testing, A/B testing, heuristic evaluation
- **Analytics**: Interpreting user behavior data, funnel analysis, heatmaps

### User Interface (UI) Design
- **Visual Design**: Typography, color theory, layout, composition, visual hierarchy
- **Design Tools**: Figma (primary), Sketch, Adobe XD, Adobe Creative Suite
- **Responsive Design**: Designing for mobile, tablet, desktop breakpoints
- **Micro-interactions**: Animation, transitions, loading states, hover effects
- **Icon Design**: Custom icon sets, icon systems, SVG optimization
- **Illustration**: Custom illustrations, brand-aligned visual assets
- **Design Tokens**: Color palettes, typography scales, spacing systems

### Interaction Design (IxD)
- **Interaction Patterns**: Understanding common UI patterns, when to use each
- **State Management**: Designing for empty states, loading states, error states, success states
- **Navigation Design**: Primary navigation, breadcrumbs, pagination, tabs
- **Forms & Input**: Form design best practices, validation, error handling, progressive disclosure
- **Feedback Mechanisms**: Toast notifications, modals, inline messages, progress indicators
- **Gestural Interfaces**: Touch gestures, swipe actions, drag-and-drop
- **Animation Principles**: Easing, timing, choreography, purposeful motion

### Design Systems
- **Component Libraries**: Building comprehensive component libraries in Figma
- **Documentation**: Component usage guidelines, do's and don'ts, accessibility notes
- **Design Tokens**: Color, typography, spacing, border radius, shadow systems
- **Component API Design**: Defining component props/variants, composition patterns
- **Versioning**: Managing design system versions, release notes, migration guides
- **Governance**: Design system review processes, contribution guidelines
- **Adoption**: Evangelizing design system, training teams, measuring adoption

## Accessibility (A11y)

### WCAG Compliance
- **WCAG 2.1 Level AA**: Understanding all WCAG guidelines, applying in designs
- **Perceivable**: Text alternatives, captions, adaptable content, color contrast
- **Operable**: Keyboard accessible, enough time, seizure prevention, navigable
- **Understandable**: Readable text, predictable behavior, input assistance
- **Robust**: Compatible with assistive technologies

### Accessible Design Practices
- **Color Contrast**: Ensuring 4.5:1 for body text, 3:1 for large text, testing with tools
- **Keyboard Navigation**: Designing for keyboard-only users, focus indicators, tab order
- **Screen Reader Support**: Semantic HTML, ARIA labels, descriptive link text
- **Touch Targets**: Minimum 44x44px touch targets, adequate spacing
- **Focus Management**: Logical focus order, focus trapping in modals, skip links
- **Alternative Text**: Writing descriptive alt text for images, decorative images
- **Motion Sensitivity**: Respecting prefers-reduced-motion, offering static alternatives

### Accessibility Testing
- **Automated Testing**: Axe, Lighthouse, WAVE for automated accessibility scanning
- **Manual Testing**: Keyboard testing, screen reader testing (NVDA, JAWS, VoiceOver)
- **Color Blindness**: Testing designs with color blindness simulators
- **Zoom Testing**: Testing at 200% zoom, ensuring text doesn't clip
- **Assistive Technology**: Testing with magnifiers, switch controls, voice control

## Design Process

### Discovery & Research
- **Stakeholder Interviews**: Understanding business goals, constraints, success metrics
- **User Research**: Identifying user needs, pain points, motivations, context of use
- **Competitive Analysis**: Analyzing competitor products, identifying best practices, gaps
- **Analytics Review**: Understanding current user behavior, drop-off points, popular features
- **Technical Constraints**: Understanding technical feasibility, API limitations, performance constraints

### Ideation & Exploration
- **Sketching**: Rapid ideation, exploring multiple concepts, low-fidelity exploration
- **Wireframing**: Creating low-fidelity wireframes, iterating quickly
- **User Flows**: Mapping out user journeys, happy paths, edge cases
- **Information Architecture**: Organizing content, navigation structure, taxonomy
- **Brainstorming**: Collaborative ideation sessions, "How Might We" questions

### Design & Iteration
- **High-Fidelity Mockups**: Creating pixel-perfect designs in Figma
- **Interactive Prototypes**: Building clickable prototypes for testing
- **Usability Testing**: Testing designs with real users, gathering feedback
- **Design Critiques**: Presenting work for feedback, iterating based on input
- **Developer Handoff**: Specs, annotations, design tokens, assets, collaboration

### Design Review & QA
- **Design QA**: Reviewing implementation against designs, pixel-perfect checks
- **Accessibility Review**: Testing implementation for accessibility compliance
- **Cross-Browser Testing**: Ensuring consistency across browsers
- **Responsive Testing**: Testing across devices, breakpoints
- **Iteration**: Addressing issues found in QA, refining based on real-world usage

- **Pagination**: Page numbers, previous/next, infinite scroll considerations
- **Search**: Search UI, autocomplete, filters, recent searches

### Form Design
- **Form Layout**: Single column vs multi-column, label placement, field grouping
- **Input Validation**: Inline validation, error message design, success states
- **Required Fields**: Indicating required fields, optional field strategy
- **Complex Inputs**: Date pickers, file uploads, multi-select, autocomplete
- **Form Progress**: Multi-step forms, progress indicators, saving draft state
- **Submit Buttons**: Button placement, loading states, success confirmation

### Feedback & Messaging
- **Toast Notifications**: Positioning, timing, actionability, priority
- **Modals**: When to use modals, close options, focus management
- **Inline Messages**: Contextual messages, banners, alerts
- **Empty States**: Helpful empty states, onboarding, clear next actions
- **Loading States**: Skeleton screens, spinners, progress bars
- **Error Handling**: User-friendly error messages, recovery actions, preventing errors

### Data Display
- **Tables**: Sortable columns, pagination, filtering, row actions, responsive tables

- **Charts & Graphs**: Choosing appropriate visualizations, tooltips, legends, accessibility

- **Real-Time Updates**: Live data, update indicators, conflict resolution

## Responsive & Mobile Design

### Mobile-First Design
- **Touch Targets**: Adequate size and spacing for thumb zones
- **Gestures**: Swipe, pinch, pull-to-refresh, long-press
- **Mobile Navigation**: Bottom nav, hamburger menu, swipeable tabs
- **Input Optimization**: Appropriate keyboard types, auto-focus, auto-complete
- **Performance**: Image optimization, lazy loading, perceived performance
- **Offline Considerations**: Offline states, caching, sync indicators

### Breakpoint Strategy
- **Mobile**: <640px, single column, stacked content
- **Tablet**: 640-1024px, 2-column layouts, collapsed nav
- **Desktop**: >1024px, multi-column layouts, expanded nav, more content density
- **Fluid Design**: Using relative units, flexible grids, avoiding fixed widths
- **Testing**: Testing at multiple breakpoints, edge cases between breakpoints

### Progressive Enhancement
- **Core Experience**: Ensuring core functionality works everywhere
- **Enhanced Features**: Adding richer interactions for capable devices
- **Feature Detection**: Checking for support before using features
- **Graceful Degradation**: Providing fallbacks for unsupported features
